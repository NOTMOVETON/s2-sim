## Компоненты

### 1. WorldSnapshot — структура снапшота мира

**Файл:** `s2_core/include/s2/world_snapshot.hpp`

WorldSnapshot — это моментальный снимок всего состояния мира в определённый момент времени. Он содержит:

- Время симуляции (sim_time)
- Информация обо всех агентах: id, name, pose, velocity, визуальное описание (тип фигуры, размеры, цвет), заряд батареи, скорость, блокировка движения
- Информация о пропах: id, тип, позиция, визуальное описание, прикреплён ли к агенту
- Информация об акторах: id, name, позиция, визуальное описание, текущее состояние
- Информация о зонах: id, включена ли, форма (центр/радиус или центр/размер), какие агенты внутри
- Статическая геометрия мира: тип фигуры, позиция, размеры, цвет — отправляется только при первом подключении клиента

### 2. Сериализация в JSON

**Файл:** `s2_core/src/world_snapshot.cpp`

Функция `snapshot_to_json()` преобразует WorldSnapshot в JSON с использованием nlohmann/json (уже в зависимостях Dockerfile).

**Правила сериализации:**
- Pose3D → объект с полями x, y, z, yaw
- Velocity → объект с vx, vy, vz, wx, wy, wz
- VisualDesc → объект с type, size (массив), color (строка-шестнадцатеричный код)
- Все ID — целые числа
- Опциональные поля → null если не заданы

**Параметр `include_geometry`:** если true — включить статическую геометрию мира в поле "geometry". По умолчанию false, true только при первом подключении клиента.

### 3. VizServer — WebSocket + HTTP сервер

**Файлы:** `s2_visualizer/src/viz_server.hpp`, `s2_visualizer/src/viz_server.cpp`

**Зависимости:**
- uWebSockets (уже в Dockerfile) — WebSocket + HTTP сервер
- nlohmann/json (уже в Dockerfile) — JSON сериализация

**Что делает VizServer:**

1. **HTTP сервер на порту 1937:**
   - Отдаёт статические файлы из `s2_visualizer/web/`
   - При запросе `/` — `index.html`
   - При запросе `/js/app.js` — JavaScript
   - Остальные файлы — по пути

2. **WebSocket сервер на порту 8765:**
   - Принимает подключения от клиентов (браузер)
   - При каждом вызове `publish(snapshot)` — отправляет JSON всем подключённым клиентам
   - При первом подключении клиента — отправляет снапшот с геометрией мира
   - Последующие обновления — без геометрии (меньше трафик)

3. **Thread safety:**
   - `start()` запускает сервер в отдельном потоке
   - `publish()` копирует снапшот в mutex-protected буфер — сервер читает из буфера
   - `stop()` завершает сервер и ждёт завершения потока

4. **Методы:**
   - `VizServer(ws_port, http_port, static_path)` — конструктор
   - `start()` — запуск
   - `stop()` — остановка
   - `publish(snapshot)` — отправка всем клиентам
   - `poll_commands()` — получение команд от визуализатора (пока заглушка, зарезервировано на будущее)

### 4. Three.js клиент

**Файлы:** `s2_visualizer/web/index.html`, `s2_visualizer/web/js/app.js`

**Зависимости:** Three.js через CDN (unpkg), OrbitControls через CDN — без локальной установки.

**index.html:**
- Минимальная HTML-страница с importmap для Three.js
- Подключает `js/app.js` как ES module
- Стили: canvas на весь экран, без margins
- Информационный блок (top-left corner) — показывает статус подключения и время симуляции

**app.js — функциональность:**

1. **Инициализация сцены:**
   - Scene с тёмным фоном
   - PerspectiveCamera
   - WebGLRenderer с antialias и shadowMap
   - OrbitControls (вращение/зум/пан мышью)

2. **Освещение:**
   - AmbientLight (мягкий общий свет)
   - DirectionalLight (основной направленный свет с тенями)

3. **Сетка:**
   - GridHelper 40×40 — визуальная сетка пола

4. **Подключение WebSocket:**
   - Подключение к `ws://hostname:8765`
   - При получении JSON — вызов `updateScene()`
   - При обрыве — автоматический reconnect с нарастающей задержкой

5. **Создание мешей:**
   - Статическая геометрия (стены) — BoxGeometry, SphereGeometry, CylinderGeometry по типу
   - Агенты — BoxGeometry (как в visual), цвет из visual.color
   - Зоны — полупрозрачный wireframe BoxGeometry или CircleGeometry на уровне z=0.05
   - Все меши сохраняются по ключам: `agent_${id}`, `prop_${id}`, `zone_${id}`

6. **Обновление позиций:**
   - При каждом снапшоте обновлять позицию и вращение каждого меша
   - Из pose.x/y/z — position, из pose.yaw — rotation.y

7. **Reconnect:**
   - При закрытии WebSocket — пробовать переподключиться через 1с, 2с, 3с... (до 5с)

### 5. Интеграция с SimEngine

**Модификация:** `sim_engine.hpp`

В SimEngine добавить:
- Указатель на VizServer (не владеет, просто ссылка)
- В `tick()`: фазу виза — аккумулировать dt, при достижении 1/viz_rate вызывать `viz_server_->publish(build_snapshot())`
- Функцию `build_snapshot()` — собирает WorldSnapshot из текущего world_

### 6. Main executable

**Файл:** `workspace/s2_core/src/main.cpp`

Программа:
1. Загружает сцену из YAML (путь из аргумента или дефолт `s2_config/scenes/test_basic.yaml`)
2. Создаёт SimEngine с конфигом из сцены
3. Создаёт и запускает VizServer
4. Привязывает VizServer к SimEngine
5. Задаёт тестовую команду первому роботу (например set_command(0.5, 0.1))
6. Запускает `engine.run()`

### 7. Обновление docker-compose.yml

Добавить сервис `sim`:
- Сборка из того же Dockerfile
- volume: ../workspace:/workspace
- Проброс портов: 1937 (HTTP) и 8765 (WebSocket)
- Команда: запуск бинарника симуляции с путём к сцене
