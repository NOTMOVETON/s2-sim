# Задача 04.10 — ColorPlugin (смена цвета робота по ROS2 сервису)

## Цель

Плагин `color` — полноценный `IAgentPlugin`, изменяющий цвет робота в симуляции
на заданное время по запросу ROS2 сервиса.

## Архитектура

`ColorPlugin` является симуляционным плагином (как `GnssPlugin`, `ImuPlugin`):
- Живёт в `s2_plugins`, работает без ROS2
- Изменяет `agent.visual.color` в `update(dt, agent)` → это HEX-поле в `VisualDesc`
- `AgentSnapshot.visual.color` уходит в JSON → Three.js `material.color` меняется мгновенно
- Сервисный вызов из ROS2 — лишь способ подать команду; таймер и логика — чисто симуляционные

При сбросе таймера восстанавливается оригинальный цвет агента из `VisualDesc.color`,
запомненный при первом вызове `update()`.

## Новый virtual метод: initialize()

Плагину нужно запомнить исходный цвет агента ДО первого `update()`. Для этого
добавляется optional метод в базовый класс:

```cpp
// В IAgentPlugin:
virtual void initialize(Agent& agent) { (void)agent; }
```

Вызывается из `SimTransportBridge::init()` после регистрации всех плагинов.

## ColorPlugin

### Интерфейс

```cpp
class ColorPlugin : public IAgentPlugin {
public:
    std::string type() const override { return "color"; }
    std::vector<std::string> service_names() const override { return {service_name_}; }

    void initialize(Agent& agent) override;
    // Запоминает agent.visual.color как default_color_ (если не задан в конфиге)

    void from_config(const YAML::Node& node) override;
    // service: /set_color
    // default_color: "#FF6B35"  (опционально; если не задан — берётся из agent.visual.color)

    std::string handle_service(const std::string& service_name,
                               const std::string& request_json) override;
    // request: {"color": "#FF0000", "duration": 3.0}
    // response: {"success": true}
    // → active_color_ = color, timer_ = duration_

    void update(double dt, Agent& agent) override;
    // timer_ > 0:  agent.visual.color = active_color_; timer_ -= dt
    // timer_ <= 0: agent.visual.color = default_color_

    std::string to_json() const override;
    // {"plugin":"color","color":"#FF0000","remaining":2.7}
    // При дефолтном: {"plugin":"color","color":"#FF6B35","remaining":0}

private:
    std::string service_name_{"/set_color"};
    std::string default_color_{"#FF6B35"};  // заполняется из agent.visual.color при initialize()
    std::string active_color_;
    double timer_{0.0};
};
```

### Цвета

Принимается любая строка, которую понимает Three.js:
- HEX: `"#FF0000"`, `"#ff0000"`
- CSS-имена: `"red"`, `"green"`, `"blue"`, `"orange"`, `"yellow"`

## Конфигурация

```yaml
agents:
  - name: dozer
    visual:
      type: box
      size: [2.5, 1.5, 1.5]
      color: "#FF6B35"     # этот цвет станет default_color_ плагина
    plugins:
      - type: color
        service: /set_color
        # default_color опциональный — если не задан, берётся из agent.visual.color выше
```

## Вызов из ROS2

```bash
# Покрасить робота в красный на 5 секунд
ros2 service call /set_color s2_msgs/srv/PluginCall \
  "{request: '{\"color\":\"#FF0000\",\"duration\":5.0}'}"

# Покрасить в синий на 2 секунды
ros2 service call /set_color s2_msgs/srv/PluginCall \
  "{request: '{\"color\":\"blue\",\"duration\":2.0}'}"

# Немедленно вернуть исходный цвет (duration: 0)
ros2 service call /set_color s2_msgs/srv/PluginCall \
  "{request: '{\"color\":\"\",\"duration\":0}'}"
```

## Тесты

**`test_color_plugin.cpp`**:
- `InitializeStoresOriginalColor` — `initialize()` запоминает `agent.visual.color`
- `ServiceCallChangesColor` — `handle_service()` + `update()` → `agent.visual.color` изменился
- `TimerExpiry` — после N тиков цвет восстановился в дефолтный
- `ZeroDurationImmediate` — `duration: 0` → цвет меняется и сразу сбрасывается
- `ToJsonContainsColor` — `to_json()` содержит текущий цвет и remaining
