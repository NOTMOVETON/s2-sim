# Фича 04.5 — Интерактивный визуализатор

## Цель
Расширить веб-визуализатор для интерактивного взаимодействия с роботом: клик по роботу, боковая панель, управление симуляцией, перемещение робота.

## Функциональность

### 4.5.1 Управление симуляцией (Play/Pause/Restart)
- Кнопки в верхней панели HTML
- **Play**: продолжить симуляцию (resume)
- **Pause**: остановить симуляцию (pause)
- **Restart**: сбросить всё к начальному состоянию и поставить на паузу
- Реализация:
  - VizServer обрабатывает POST /command или GET /command?cmd=...
  - SimEngine: методы pause(), resume(), reset()
  - При reset: переустановить все world_pose/velocity из начальных значений
  - Нужно хранить `initial_state` каждого агента при загрузке

### 4.5.2 Клик по роботу
- Raycaster в Three.js при клике на меш агента
- При клике — открытие боковой панели слева
- При клике вне робота — закрытие панели

### 4.5.3 Боковая панель робота
**Метаинформация:**
- ID агента
- Имя
- Подключённые плагины (список типов)

**Текущее состояние:**
- Позиция: x, y, z, yaw
- Скорость: vx, vy, wz
- Обновляется в реальном времени из SSE

**Режим слежения:**
- Кнопка "Follow"
- Камера привязана к роботу с offset
- Можно крутить (OrbitControls) и зумить вокруг робота
- Кнопка "Unfollow" для отключения

**Выходы плагинов:**
- JSON-панель с последними данными от каждого плагина
- Пример:
  ```json
  {
    "gnss": {"lat": 55.75, "lon": 37.62, "accuracy": 0.5},
    "imu": {"gyro_z": 0.3, "accel_x": 0.1, "yaw": 0.15}
  }
  ```

### 4.5.4 Перемещение робота мышкой
- При клике на робота — появляется система координат (3 стрелки)
- TransformControls из Three.js addons
- Режимы: Translate (вдоль осей X/Y) или Rotate (вокруг Z)
- При манипуляции — данные отправляются на сервер
- Сервер обновляет позицию агента

### 4.5.5 SSE данные для интерактивности
Расширить формат снапшота:
```json
{
  "agents": [...],
  "props": [...],
  "actors": [...],
  "plugins": {
    "agent_0": {
      "gnss": {"lat": 55.75, ...},
      "imu": {"gyro_z": 0.3, ...}
    }
  },
  "sim_time": 5.2,
  "paused": false
}
```

## Техническая реализация

### Серверная часть (C++)
1. Добавить POST /command endpoint в VizServer
   - `pause` — остановить tick
   - `resume` — продолжить tick
   - `reset` — сбросить состояние
   - `reset_and_resume` — сбросить и продолжить
   - `move_agent` — переместить агента (id, x, y, yaw)
2. Добавить поле `paused_` в SimEngine
3. Добавить метод `reset()` — переустановка из начального состояния
4. Добавить `get_agent_initial_state(id)` — получение начальной позиции

### Клиентская часть (JS)
1. Добавить HTML кнопки Play/Pause/Restart
2. Добавить Raycaster для клика по мешам агентов
3. Создать боковую панель с секциями
4. Добавить TransformControls для перемещения
5. Подписка на SSE и обновление данных панели

Добавить поля:
