# Фича 04.6 — Управление плагинами из симуляции
**Статус: ✅ Реализовано**
## Зачем

Позволить пользователю отправлять команды плагинам (входные данные) через UI визуализатора.
Это открывает возможность интерактивного управления: установка скорости, целевых позиций,
включение/выключение режимов и т.д.

## Концепция

У каждого плагина может быть **входы** (inputs). Если плагин принимает управление — он объявляет
свои входы, а UI визуализатора рендерит форму для этих входов.

## Архитектура

### 1. Расширение IAgentPlugin

Добавить методы для работы с входами:

```cpp
class IAgentPlugin {
public:
    virtual ~IAgentPlugin() = default;
    virtual std::string type() const = 0;
    virtual void update(double dt, Agent& agent) = 0;
    virtual void from_config(const YAML::Node& node) = 0;
    virtual std::string to_json() const = 0;

    // Новые методы:
    virtual bool has_inputs() const { return false; }
    virtual std::string inputs_schema() const { return ""; }  // JSON Schema входов
    virtual void handle_input(const std::string& json_input) {}
};
```

### 2. Схема входов (JSON Schema)

Каждый плагин описывает свои входы через JSON Schema:

```json
{
  "linear_velocity": {"type": "number", "default": 0, "min": -2, "max": 2},
  "angular_velocity": {"type": "number", "default": 0, "min": -1.5, "max": 1.5}
}
```

### 3. Протокол отправки

```
POST /command?cmd=plugin_input&agent_id=0&plugin=diff_drive
Body: {"linear_velocity": 1.0, "angular_velocity": 0.5}
```

### 4. UI визуализатора

- В боковой панели рядом с каждым плагином есть кнопка "Отправить" (или иконка)
- Открывается модальное окно или inline-форма со схемой входов
- Пользователь заполняет поля, нажимает "Send"
- Отправка на сервер через POST /command
- Плагины без входов — кнопка не показывается

## Пример для DiffDrivePlugin

Входы:
```json
{
  "linear_velocity": {"type": "number", "default": 0.0, "min": -2.0, "max": 2.0, "unit": "m/s"},
  "angular_velocity": {"type": "number", "default": 0.0, "min": -1.5, "max": 1.5, "unit": "rad/s"}
}
```

При отправке:
```json
{"linear_velocity": 0.5, "angular_velocity": 0.3}
```

DiffDrivePlugin устанавливает целевые скорости в SharedState.

## Этапы реализации

### Этап 1: Бэкенд
1. Расширить `IAgentPlugin` — `has_inputs()`, `inputs_schema()`, `handle_input()`
2. Добавить `handle_plugin_input()` в SimEngine
3. Добавить `plugin_input` команду в VizCommandHandler
4. Реализовать для DiffDrivePlugin

### Этап 2: Frontend
1. Показать схему входных данных плагина в UI (вместо/рядом с выводом)
2. Рендер формы по JSON Schema
3. Отправка через POST /command?cmd=plugin_input

### Этап 3: UI улучшение панели плагинов
1. Аккордеон для плагинов (каждый плагин — строка с раскрытием)
2. Внутри — данные плагина + форма входа (если есть)

## Trade-offs

| Подход | Плюсы | Минусы |
|--------|-------|--------|
| JSON Schema | Универсально, легко парсить UI | Сложно для сложных типов |
| Формы ввода | Быстро для известных плагинов | Не масштабируется |
| Гибридный | Комбинация обоих | Больше кода |

## Ограничения

- Не все плагины имеют входы (GNSS — только output, IMU — только output)
- Входы опциональны — если плагин не реализует, просто не показывается форма
- Безопасность — валидация входов на стороне сервера обязательна
