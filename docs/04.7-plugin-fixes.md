# Задача 04.7 — Исправления багов управления плагинами

## Цель

Исправить критические баги в функциональности управления плагинами из визуализатора обнаруженные при тестировании фичи 04.6.

## Проблема 1: Конфликт ID форм плагинов (КРИТИЧНО)

**Файл:** `workspace/s2_visualizer/web/js/app.js`

**Симптом:** При отправке данных плагина для robot_1 через веб-интерфейс управление получают оба робота (или robot_0).

**Детали:**
- При клике на кнопку формы плагина создаётся HTML с `id="plugin-form-diff_drive"`
- Когда у robot_0 И robot_1 есть плагин diff_drive — оба создают элементы с одинаковым ID
- `document.getElementById("plugin-form-diff_drive")` всегда возвращает первый элемент (robot_0)
- Аналогичный конфликт для ID input-полей (частично исправлено — добавлен agentId)

**Решение:**
1. Изменить ID: `id="plugin-form-${pluginName}"` → `id="plugin-form-${agentId}-${pluginName}"`
2. Обновить `showPluginInputForm()` для поиска по новому ID
3. Убедиться что при смене агента старые формы корректно удаляются

## Проблема 2: Персистентность скорости (АРХИТЕКТУРНЫЙ БАГ)

**Файл:** `workspace/s2_plugins/include/s2/plugins/diff_drive.hpp`

**Симптом:** После отправки команды скорости робот продолжает двигаться бесконечно.

**Механизм:**
```
handle_input() → external_linear_velocity_ = 1.0, has_external_input_ = true
    ↓
update() → применяет external, current_data.desired_linear = 1.0 → SharedState
    ↓
Следующий тик: dd->desired_linear = 1.0 из SharedState
    ↓
update() → берёт из SharedState → снова применяет → бесконечный цикл
```

**Решение (вариант А — минимальный):**
Не записывать external_values в current_data.desired_*. При наличии external_input использовать их напрямую не записывая в SharedState.

**Решение (вариант Б — правильный):**
Сделать external_input "одноразовым" — применять его один раз и сбрасывать. Для постоянного движения UI должен периодически отправлять команды.

## Проблема 3: Debug-логирование (КОД-СТИЛЬ)

**Файлы:** `sim_engine.hpp`, `diff_drive.hpp`

**Описание:** Многочисленные `std::cout` логи добавленные для диагностики засоряют вывод симулятора.

**Решение:** Удалить все debug-логи.

## Критерии приёмки

1. Отправка команды robot_1 двигает только robot_1
2. После одной команды робот двигается ограниченное время (или один тик)
3. Консоль симулятора не содержит debug-логов
4. Все тесты проходят
