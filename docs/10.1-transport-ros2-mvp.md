## Что было сделано

### 1. TripleBuffer
`workspace/s2_core/include/s2/triple_buffer.hpp` — thread-safe обмен данными между sim и transport потоками.
6 тестов, включая многопоточный.

### 2. s2_transport модуль
- Stub-режим (без ROS2) — собирается всегда, `ros2_transport_stub.cpp`
- Полная реализация (`-DS2_WITH_ROS2=ON`) — `ros2_transport.cpp`

### 3. ROS2Transport — архитектура изолированных доменов
Каждый `register_agent_cmd_vel(agent_id, domain_id)`:
- Создаёт `rclcpp::Context` с `InitOptions::set_domain_id(domain_id)`
- Создаёт `rclcpp::Node` с `NodeOptions::context(context)`
- Создаёт `SingleThreadedExecutor` с `ExecutorOptions::context = context`
- Подписывается на `/cmd_vel` в этом контексте

`start()` — один `std::thread` на каждый domain_id, `spin_some(10ms)` в цикле.
`stop()` — join всех потоков, `context->shutdown()` для каждого домена.

**Без глобального `rclcpp::init()`** — транспорт сам управляет контекстами.

### 4. Dockerfile.ros2
Multi-stage: Ubuntu Noble (24.04) + ROS2 Jazzy Jalisco.
Пакеты: `ros-jazzy-ros-base`, `ros-jazzy-geometry-msgs`, `ros-jazzy-rmw-fastrtps-cpp`, etc.

### 5. Интеграция в main.cpp
```cpp
// Регистрация агентов (до start())
for (const auto& agent : world.agents()) {
    transport.register_agent_cmd_vel(agent.id, agent.domain_id);
}

// Callback → SimEngine
transport.set_cmd_callback([&engine](const s2::CmdVelCommand& cmd) {
    if (cmd.has_command) {
        engine.handle_plugin_input(cmd.agent_id, "diff_drive",
            "{\"linear_velocity\":" + std::to_string(cmd.linear_velocity) + ...);
    }
});

transport.start();
```

### 6. Сцена test_ros2.yaml
3 робота с domain_id 0, 1, 2. Запускается через `sim_ros2` в docker-compose.

## Исправленные баги

### Баг 1: Мерцание скорости (velocity flickering)
- **Симптом:** 30 Гц publish, 100 Гц sim → скорость мерцает 0↔target
- **Причина:** `has_external_input_ = false` в `update()` → команда одноразовая
- **Исправление:** Убрали сброс флага. Latch-поведение — команда сохраняется до явного {0,0}
- **Файл:** `workspace/s2_plugins/include/s2/plugins/diff_drive.hpp`

### Баг 2: Domain isolation не работала
- **Симптом:** Все роботы реагируют на domain 0 `/cmd_vel`
- **Причина:** `rclcpp::Node` без изолированного контекста → все в global context (domain 0)
- **Исправление:** Отдельный `rclcpp::Context` per domain, executor привязан через `ExecutorOptions::context`
- **Файл:** `workspace/s2_transport/src/ros2_transport.cpp`, `.hpp`

### Баг 3: Топик с префиксом → без префикса
- **Было:** `/{agent_name}/cmd_vel` (несовместимо с реальными Nav2 стеками)
- **Стало:** `/cmd_vel` в изолированном домене
