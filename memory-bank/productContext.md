# productContext.md

> **Зачем нужен этот файл:** объяснить, зачем проект существует, какую проблему решает, для кого создан и что считается успехом. Позволяет новому агенту быстро понять мотивацию проекта и не предлагать решения, противоречащие целям. Обновляется при изменении целевой аудитории, критериев успеха или контекста использования.

## Зачем существует проект

- Нужна проверка fleet management, миссий и высокоуровневой логики без тяжёлой физической симуляции.
- Нужна среда, которая масштабируется до десятков и сотен агентов.
- Нужна воспроизводимость, headless-режим и понятная отладка.
- Существующие решения (Isaac Sim, Gazebo) требуют GPU, избыточны и не масштабируются за пределы 3 роботов.

## Какой опыт должен давать проект

- Сцену можно быстро поднять в Docker без установки чего-либо локально.
- Состояние мира можно понять по docs, Memory Bank и визуализатору.
- После перезапуска сессии работа продолжается без потери контекста.
- Принятые решения фиксируются в документации, а не только в переписке.

## Пользователи проекта

- Разработчики fleet management систем — тестируют миссии и coordination.
- Разработчики высокоуровневого стека робота — проверяют Nav2, MoveIt2, custom logic.
- Интеграторы — верят, что симуляция и реальный робот ведут себя одинаково с точки зрения протокола.

## Что считается успехом

- Симуляция повторяема: одинаковый сценарий даёт одинаковый результат.
- Поведение понятно по документам и состоянию мира.
- Изменения не требуют знания локальной машины — всё в Docker.
- Новый агент может войти в проект через Memory Bank и docs без контекста из чата.

## Критерии полезности

- Симуляция не зависит от GPU и тяжёлой физической симуляции.
- ROS — только транспортный адаптер, не ядро.
- Визуализатор — отдельный процесс, браузерный, не RViz.
- Документация живёт в файлах, а не в чате.
