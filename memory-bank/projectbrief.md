# projectbrief.md

> **Зачем нужен этот файл:** определить цели проекта, его границы и базовые требования. Любой новый агент читает этот файл первым, чтобы быстро понять, что такое S2, что он делает и что — нет. Обновляется только при изменении целей или границ проекта.

## Кратко

S2 — **семантический / high-level interaction simulator** для тестирования высокоуровневой робототехнической логики: fleet management, миссии, взаимодействие с инфраструктурой.

## Что S2 делает

- Принимает управляющие сигналы от стека каждого робота через транспортный плагин.
- Обновляет состояние всех сущностей каждый тик (кинематика, без физики).
- Применяет эффекты среды к движению и сенсорам.
- Вычисляет эффективные ограничения из нескольких источников (resolver).
- Публикует сенсорные данные каждому роботу.
- Содержит интерактивные объекты (двери, лифты, маркеры).
- Содержит динамические препятствия (пешеходы).
- Масштабируется до 100 агентов.

## Что S2 НЕ делает

- Нет физической симуляции (трение, инерция, столкновения с импульсом).
- Нет рендера изображений.
- Нет привязки к ROS (ROS — один из транспортных адаптеров).
- Нет распознавания образов, камер, компьютерного зрения.
- Нет контактной механики захвата.

## Ключевое упрощение

Всё взаимодействие робота с миром — высокоуровневые факты:
- Робот не «видит ArUco маркер камерой» — он «оказался рядом и получил позицию».
- Робот не «нажимает кнопку двери» — он «отправил команду и ждёт смены состояния».
- Робот не «физически толкает ящик» — он «отправил команду захвата и ящик прикрепился».

## Границы проекта

- Делает: кинематическое обновление мира, агенты, акторы, зоны, эффекты, сенсоры, транспорт и визуализатор.
- Не делает: физику импульсов, рендер изображений, компьютерное зрение и ROS как зависимость ядра.
- Работает детерминированно в одном потоке симуляции.
